ian.Location

ian.Location(window: Window?)

This core component acts as a shim for missing History Manipulation API implementations in older environments. The core idea is that location hash parts are merged with the location pathname parts. When used together with an ian.History object on the same host Window object, any path-related values provided by both components should match.

The signature of the hashes expected by the component is #!path:state_index which is the usual hashbang format with the added postfix required for correct behavior in case of older environments.

The controlled Window object is observed for hashchange and popstate events (or periodically checked if such events are not available in the host environment).

Its main usage is in cooperation with a routing–ian.Router–object.

Basic Setup

var location = new ian.Location();

By default, the component uses the global Window object.
